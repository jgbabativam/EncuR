3.7 Transformación y resumen de datos con mutate, summarise y group_by

Las funciones mutate(), summarise() y group_by() forman parte del ecosistema tidyverse, y son herramientas fundamentales para la manipulación y análisis de datos provenientes de encuestas de hogares.

La función mutate() permite crear nuevas variables o modificar las existentes dentro de una base de datos. En el contexto de las encuestas de hogares, es común generar variables nuevas variables que faciliten el análisis, como indicadores de pobreza, necesidades básicas, entre otros.

Por ejemplo, a partir de la base datablue, se puede crear una nueva variable denominada ingreso2, que representa el doble del ingreso individual:

datablue2 <- datablue %>% 
             mutate(ingreso2 = 2 * ingcorte) %>% 
             head()

Es posible crear varias variables simultáneamente, ya que mutate() reconoce de forma secuencial las variables que se van generando dentro de la misma instrucción. En el siguiente ejemplo se crean dos nuevas variables en la base de datos:

datacat <- datablue %>% 
           mutate(ingreso2 = 2 * ingcorte,
                  ingreso4 = 2 * ingreso2) %>% 
           head()

La función summarise() (o summarize()) se utiliza para resumir información. A diferencia de mutate(), que agrega nuevas columnas al conjunto de datos, summarise() colapsa filas y devuelve un nuevo conjunto de datos con resultados agregados, como promedios, totales o medidas de dispersión.

Por ejemplo, podría usarse para calcular el ingreso promedio o el número de observaciones dentro de un grupo específico. Sin embargo, para que el resumen tenga un sentido analítico, suele combinarse con la función group_by().

La función group_by() permite definir grupos dentro del conjunto de datos en función de una o más variables categóricas, como región, sexo o área geográfica. Una vez definidos los grupos, es posible aplicar funciones de resumen como summarise() o de transformación como mutate() para obtener resultados por grupo.

Por ejemplo, el siguiente código muestra cómo generar el número de encuestas efectivas realizadas en cada uno de los estados de Brasil, ordenadas de mayor a menor al usar arrange():

data2 %>% 
  group_by(estados) %>% 
  summarise(n = n()) %>% 
  arrange(desc(n)) %>% 
  head()
## # A tibble: 6 × 2
##   estados          n
##   <fct>        <int>
## 1 SaoPaulo     40008
## 2 MinasGerais  32933
## 3 RioGrandeSur 26259
## 4 Bahia        26155
## 5 RioJaneiro   25858
## 6 Para         22489

De forma similar, se puede calcular el número de encuestas por sexo del respondiente:

data2 %>% 
  group_by(sexo) %>% 
  summarise(n = n()) %>% 
  arrange(desc(n)) 
## # A tibble: 2 × 2
##   sexo        n
##   <fct>   <int>
## 1 Mujer  183681
## 2 Hombre 173223

O en caso de necesitar la consulta del número de encuestas efectivas por área geográfica, se realizaría de la siguiente manera:

data2 %>% 
  group_by(area_ee) %>% 
  summarise(n = n()) %>% 
  arrange(desc(n))
## # A tibble: 2 × 2
##   area_ee          n
##   <fct>        <int>
## 1 Area urbana 304564
## 2 Area rural   52340

Finalmente, una consulta muy común en las encuestas de hogares es conocer la distribución de las entrevistas según el parentesco con el jefe del hogar (jefe, cónyuge, hijos, etc.):

data2 %>% 
  group_by(paren_ee) %>% 
  summarise(n = n()) %>% 
  arrange(desc(n)) 
## # A tibble: 6 × 2
##   paren_ee                n
##   <fct>               <int>
## 1 Hijos              126206
## 2 Jefe               117939
## 3 Cónyuge             73725
## 4 Otros parientes     36508
## 5 Otros no parientes   2342
## 6 Servicio doméstico    184
